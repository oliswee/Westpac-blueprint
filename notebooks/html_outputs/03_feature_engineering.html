Notebook 03 — Feature Engineering & Dimension Validation¶

Westpac Blueprint — Exploration Phase

Objective¶

This is the analytical core of the exploration phase.

The product team needs to know: if we build a Readiness Score, which dimensions should it track?

We do NOT build the scoring engine here. We validate whether the 6 candidate dimensions actually differentiate customers who get approved for loans vs those who don't — using statistical tests and stratification.

Output: Evidence-based feature selection for the product's Readiness Score.

# Load features and build age for cohort split
features = pd.read_parquet(PROC_FEATURES)

features_w_age = features.copy()
features_w_age['AGE_YEARS'] = master['DAYS_BIRTH'].abs() / 365.25
features_w_age['IS_YOUNG'] = (features_w_age['AGE_YEARS'] < 35).astype(int)

validation_results = []
feature_cols = [c for c in features.columns
                if c not in ('SK_ID_CURR', 'TARGET', 'AGE_YEARS', 'IS_YOUNG', 'CNT_CHILDREN',
                             'composite_score', 'tier')]

for col in feature_cols:
    # Test 1: Default vs Non-Default
    default_vals = features_w_age.loc[features_w_age['TARGET'] == 1, col].dropna()
    no_default_vals = features_w_age.loc[features_w_age['TARGET'] == 0, col].dropna()
    if len(default_vals) > 30 and len(no_default_vals) > 30:
        _, p_default = stats.ttest_ind(default_vals, no_default_vals, equal_var=False)
    else:
        p_default = 1.0

    # Test 2: Young vs Mature
    young_vals = features_w_age.loc[features_w_age['IS_YOUNG'] == 1, col].dropna()
    mature_vals = features_w_age.loc[features_w_age['IS_YOUNG'] == 0, col].dropna()
    if len(young_vals) > 30 and len(mature_vals) > 30:
        _, p_age = stats.ttest_ind(young_vals, mature_vals, equal_var=False)
    else:
        p_age = 1.0

    # Determine group
    for group_name, group_cols in ALL_FEATURE_GROUPS.items():
        if col in group_cols or col.startswith(group_name.split('_')[0]):
            dim_group = group_name
            break
    else:
        dim_group = 'other'

    validation_results.append({
        'feature': col,
        'dimension': dim_group,
        'p_default': p_default,
        'p_age': p_age,
        'sig_default': p_default < 0.05,
        'sig_age': p_age < 0.05,
    })

val_df = pd.DataFrame(validation_results)
print(f'Features tested: {len(val_df)}')
print(f'Sig. for default:    {val_df["sig_default"].sum()} features')
print(f'Sig. for age cohort: {val_df["sig_age"].sum()} features')

    n_features=('feature', 'count'),

5. ANOVA: Are the 6 Dimensions Collectively Significant?¶

# One-way ANOVA for each score component by default status
score_cols = [
    'income_stability_score', 'spending_discipline_score',
    'savings_consistency_score', 'debt_burden_score',
    'asset_signal_score', 'behavioral_score'
]

print('One-way ANOVA: default vs non-default for each dimension score\n')
anova_results = []
for col in score_cols:
    if col not in features_w_age.columns:
        continue
    groups = [
        features_w_age.loc[features_w_age['TARGET'] == 0, col].dropna(),
        features_w_age.loc[features_w_age['TARGET'] == 1, col].dropna(),
    ]
    f_stat, p_val = f_oneway(*groups)
    sig = '***' if p_val < 0.001 else '**' if p_val < 0.01 else '*' if p_val < 0.05 else 'ns'
    anova_results.append({'dimension': col.replace('_score', '').replace('_', ' ').title(),
                          'F_statistic': f_stat, 'p_value': p_val, 'sig': sig})

anova_df = pd.DataFrame(anova_results).sort_values('F_statistic', ascending=False)
display(anova_df.style.background_gradient(cmap='Blues', subset=['F_statistic'])
        .format({'F_statistic': '{:,.1f}', 'p_value': '{:.2e}'})
        .set_caption('ANOVA: Dimension Scores by Default Status'))

young_scores = {}

8. Conclusion: Dimension Validation Summary¶

The analysis provides evidence that all 6 candidate dimensions have statistically significant discriminative power — some stronger than others.

Dimension Discriminative? Recommendation for Product
Savings Consistency Strongest Must include in Readiness Score
Debt Burden Strong Must include — core credit signal
Spending Discipline Moderate-Strong Include — differentiates young cohort well
Income Stability Moderate Include — baseline affordability signal
Behavioral Moderate Include — captures broker vs direct patterns
Asset Signal Weaker Supplementary — more relevant for loan sizing than readiness

For the product team: These 6 dimensions are validated by data as meaningful differentiators of loan outcomes. The Blueprint Readiness Score should monitor these — with savings consistency and debt burden weighted most heavily.
